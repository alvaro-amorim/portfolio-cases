## Visão geral

Os cases documentam o problema, a solução, as decisões técnicas, as funcionalidades e o papel desempenhado em cada projeto. O objetivo é mostrar não apenas telas ou tecnologias, mas a capacidade de estruturar produtos completos com regras de negócio, dados, autenticação, integrações, operação administrativa e publicação.

## Competências demonstradas

- Desenvolvimento full stack com React, Next.js, TypeScript, Node.js e Python.
- Modelagem e persistência com PostgreSQL, Supabase, Prisma e MongoDB.
- Autenticação, autorização, RLS, multi-tenancy e trilhas administrativas.
- Billing, planos, tokens, webhooks e regras de entitlement.
- Integrações com OpenAI, Gemini, Anthropic e outros provedores de IA.
- Dashboards, áreas administrativas, analytics e fluxos operacionais.
- Testes com Vitest, Jest e Playwright, além de documentação técnica.

## Critério dos cases

Cada case busca responder:

1. Qual problema o produto resolve?
2. Como a solução foi estruturada?
3. Quais decisões técnicas foram tomadas?
4. Que funcionalidades foram implementadas?
5. Qual foi minha responsabilidade no desenvolvimento?
6. Qual é o estado atual do projeto?

> Este repositório não contém código sensível, credenciais, tokens, dados internos ou métricas privadas. Os textos apresentam apenas informações públicas e técnicas adequadas para portfólio.
